# 設計：画面イメージとユーザーストーリー

## 1. ユーザーストーリー

「ユーザーとして、〇〇したい。なぜなら△△だから」の形で、この仕組みが満たすべき体験を整理する。

| # | ストーリー | 優先度 |
|---|-----------|--------|
| US1 | 朝にダッシュボードを開くと、昨日の日本人メジャーリーガー全員の試合結果がひと目で分かる | 高 |
| US2 | 投手と野手が分かれて表示され、それぞれに合った指標（投手＝防御率/奪三振 等、野手＝打率/本塁打 等）が見える | 高 |
| US3 | 「更新」ボタンを押すと、最新の成績に更新される | 高 |
| US4 | 試合がなかった選手は「登板なし／出場なし」とハッキリ分かる | 高 |
| US5 | 昨日活躍した選手（本塁打・好投など）が強調表示され、すぐ目に入る | 中 |
| US6 | 各選手の「昨日の試合」と「シーズン通算」を並べて比較できる | 高 |
| US7 | 過去の日付の結果も後から振り返れる（データが日付ごとに溜まっていく） | 中 |
| US8 | スマホでもPCでも崩れずに見られる | 中 |
| US9 | （将来）毎朝決まった時間に自動で最新化される | 低（後で追加） |

### 画面の構成要素

- **ヘッダー**：タイトル、最終更新日時（日本時間）、更新ボタン。
- **昨日のハイライト**：昨日試合があった選手の結果を1行ずつ要約。活躍には 🔥 マーク。
- **投手セクション**：選手カードを並べる。各カードに「昨日の試合」「シーズン通算」を上下に表示。
- **野手セクション**：同上（項目は野手向け）。
- **大谷選手**：野手セクションに表示しつつ、投球登板があった日は投手セクションにも登場。
- **今シーズン成績が無い選手は非表示**（昇格前・故障者リスト中など）。成績が出れば自動で表示される。

### 強調・状態の表現（US4・US5）

- 昨日**試合あり＋活躍**：カードを明るく、🔥アイコン。
- 昨日**試合あり（平凡）**：通常表示。
- 昨日**試合なし**：「登板なし／出場なし」とグレー表示。

## 3. 操作の流れ（ユーザー目線）

1. ブラウザでダッシュボードを開く。
2. 前回取得したデータがすぐ表示される（待たされない）。
3. 最新にしたいときは「🔄 更新」ボタンを押す → 数秒で最新成績に切り替わる。
4. 上から「ハイライト → 投手 → 野手」と眺めるだけで、昨日の全体像と通算成績が把握できる。
