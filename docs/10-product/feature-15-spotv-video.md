# ⑮ SPOTV NOW ダイジェスト動画の埋め込み（実装メモ）

SPOTV NOW（日本のスポーツ配信サービス）公式YouTubeが出している「選手別のダイジェスト動画」を、
選手の推移ページに埋め込んで再生できるようにした機能。2026-06-05 実装。

## 何ができるか

- 選手の推移ページ（`ohtani.html?id=選手ID`）の上部に「🎬 SPOTV NOW ダイジェスト動画」を表示。
- そこに、その選手の**SPOTV NOW公式の再生リスト**を埋め込み、最新のダイジェストから再生できる。
- 対応表に登録がある選手だけ表示。登録が無い選手はセクションごと隠す（レイアウトは崩れない）。

## なぜこの作りにしたか（設計判断）

- **動画1本ごとのIDは試合のたびに変わる**ので、固定はできない。一方で**選手別の「再生リスト」IDは固定**。
- **置き場所は推移ページ**にした。一覧ページ（index）に動画を何個も埋め込むと重くなるが、
  推移ページは1人ぶんなので埋め込み1個で済み、軽い。選手専用ページという性格にも合う。
- **プライバシー配慮**で `youtube-nocookie.com` の埋め込みを使用。

## 最新動画の取得ロジック（2026-06-05 改修）

当初は「再生リストをまるごと埋め込み」していたが、**一部の選手（今永・鈴木）は選手別の再生リストがSPOTV NOW側で更新されず古いまま**になる問題があった（並び順の問題ではなく、リストの中身自体が古い）。

そこで「**各選手の最新ダイジェスト動画を自動で探して、その1本をピンポイントで埋め込む**」方式に変更：

1. **毎朝の自動更新（fetch_stats.py）が動画を探す**（APIキー不要・YouTubeの公開フィードRSSのみ使用）。
   - SPOTV NOW公式チャンネルの**最新アップロード一覧**（`feeds/videos.xml?channel_id=…`、新しい順 約15件）から、
     **「選手名＋ダイジェスト」を含む（#shortsは除く）**最新動画を拾う。← これが要点。チャンネル全体のフィードは常に新しい。
   - 加えて選手別の再生リストフィードの中の「ダイジェスト」も候補に入れる（フィードから漏れた選手の保険）。
   - **前回保存した動画とも比較し、最も新しいものを採用**（＝一度拾った最新より古いものには戻さない＝後退防止）。
2. 結果を `data/spotv_videos.json` に保存（選手ID→{videoId, title, published}）。毎朝のコミットで反映。
3. 画面（ohtani.js）は `data/spotv_videos.json` を読み、その選手の最新動画1本を `embed/動画ID` で表示。
   - 動画情報が無い選手は、従来どおり**再生リストまるごと**にフォールバック。

### この方式の特徴・限界
- ✅ APIキー・費用なし。今の毎朝の自動更新にそのまま乗る。
- ✅ チャンネル全体フィードを見るので、選手別リストが更新されない選手でも最新が取れる。
- ⚠️ 動画が新しくなるのは**毎朝の自動更新のタイミング**（リアルタイムではない）。「🔄更新」ボタンは成績のみで動画は変えない（YouTubeフィードはブラウザから直接取れない＝CORS制限のため）。
- ⚠️ 登板間隔が空く投手などで、最後の登板動画がチャンネルフィード15件から押し出され、かつ再生リストも古い場合、一時的に最新を取り逃すことがある。その時は前回拾った動画を表示し続ける（空にはならない）。次に試合へ出れば自動で最新化。
  - 導入時（2026-06-05）、今永のみフィード15件から既に押し出されていたため、最新ダイジェスト（5/30, `D1rua_zgP78`）を `data/spotv_videos.json` に手動で初期登録した。

## 関連コマンド
- `python fetch_stats.py` … 通常の全成績取得。最後にSPOTV動画情報も更新する。
- `python fetch_stats.py spotv` … 成績は取らず、**SPOTV動画情報だけ**を更新（既存の `data/latest.json` の選手を使用）。動作確認や手動更新に便利。

## 対象ファイル

- `config/spotv_playlists.json` … 選手ID(MLB) → SPOTV NOWの再生リストID の対応表。
- `data/spotv_videos.json` … 自動生成。選手ID → 最新ダイジェスト動画（videoId/title/published）。毎朝の更新で書き換わる。
- `fetch_stats.py` … 最新動画を探す処理を追加（`build_spotv_videos` ほか）。`spotv` サブコマンドも追加。
- `ohtani.html` … 「🎬 SPOTV NOW ダイジェスト動画」セクション（`#spotv`）を追加。`ohtani.js`/`ohtani.css` の `?v=` を更新。
- `ohtani.js` … `setupSpotvVideo()`。最新動画(1本)を優先表示し、無ければ再生リストにフォールバック。無ければ隠す。
- `ohtani.css` … 動画セクションのスタイル（16:9を保って画面幅に伸縮する枠）。

## 対応表（config/spotv_playlists.json）の使い方

```json
{
  "_comment": "選手ID → SPOTV NOWの選手別 再生リストID。中身は自動更新される。",
  "660271": "PLDfB2rHFTxIssxn0c3Ccvh6zw0xQKxYOq"
}
```

- キー＝MLBの選手ID（数字）。値＝YouTube再生リストID（`list=` の後ろの文字列）。
- **選手を増やすには**：YouTubeでSPOTV NOW公式チャンネル → 再生リスト → その選手のリストを開き、
  URLの `list=` の後ろをコピーして、選手IDをキーに追加するだけ。
- ⚠️ JSONなので、説明文の中に半角ダブルクォート `"` を書かないこと（壊れる）。

## 注意・今後

- 再生リストが**全選手分あるとは限らない**（人気選手中心）。無い選手は、将来 YouTube検索での自動取得（案B）を検討。
- 再生リストの**並び順**はSPOTV NOW側の設定次第。最新が先頭に来ない場合は対応を再検討する。
- 動画は**SPOTV NOWの著作物**。公式が許可する埋め込みプレーヤーの範囲で表示し、ダウンロード・再配布はしない。
