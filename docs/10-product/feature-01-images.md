# ① 画像（選手の顔写真）詳細要件

[改修計画（ロードマップ）](roadmap.md) の項目① の詳細。2026-06-02 にユーザーと要件を確定。

## やること（確定事項）

- **選手の顔写真**を画面に追加する。チームロゴは今回は入れない（将来検討）。
- 置き方は **カード左に丸い顔写真＋名前**（投手カード・野手カード共通）。
- 画面上部の **「ハイライト」欄にも、選手名の横に小さい顔写真**を出す。

## どこから画像を取ってくるか

- MLB が公開している顔写真URLを使う。**登録・APIキー・追加データ取得は不要**。
- URL の形（選手のID番号を差し込むだけ）:
  - `https://midfield.mlbstatic.com/v1/people/{選手ID}/spots/120`
  - `spots/120` は小さめの顔写真。丸く切り抜いて表示する（CSSの円形マスク）。
- **選手のID番号は、すでに保存データ `data/latest.json` の各選手 `id` に入っている**ため、
  データの取り直しや fetch_stats.py の変更は不要。**画面側（app.js / style.css）の変更だけで実現できる。**

## 表示の仕様

### 選手カード（投手・野手）
- カード見出し（card-head）の左に、丸い顔写真を置く。右に従来どおり名前＋チーム略称。
- 顔写真サイズの目安: 直径 40〜48px 程度。
- 「🔥（活躍）」表示や成績欄は従来どおり。

### ハイライト欄
- 各行の選手名の左に、小さい丸い顔写真（直径 20〜24px 程度）を置く。

### 画像が出せないとき（フォールバック）
- 通信失敗・写真が存在しない選手の場合、画像が「壊れたアイコン」にならないよう、
  読み込み失敗時は同じ大きさの**グレーの丸**（または頭文字）に差し替えてレイアウトを崩さない。
- 実装方法: `<img>` の `onerror` で代替表示に切り替える。

## さわるファイル（予定）

- `app.js` … カード描画（pitcherCard / batterCard）とハイライト描画（renderHighlights）に `<img>` を追加。読み込み失敗時の差し替え処理。
- `style.css` … 丸い顔写真のスタイル（円形・サイズ・並び）。
- ※ `fetch_stats.py` と `data/` は**変更不要**（選手IDは既に保存済み）。

## 確認事項

- 顔写真は MLB 公式の公開URLを参照（自分のリポジトリに画像ファイルは保存しない）。
  画像はブラウザが表示時にMLBから直接読み込む。

## 実装メモ（2026-06-02 実装・確認済み）

- 画像の持ち方は **方式A（外部URL）** を採用。リポジトリに画像ファイルは保存しない。
- 顔写真URL: `https://midfield.mlbstatic.com/v1/people/{選手ID}/spots/120`
- 失敗時フォールバックは、コード内で生成する灰色の丸（インラインSVGのdata URI）。追加ファイルなし。
- `app.js`: `headshotUrl` / `avatar` / `avatarFallback` を追加。投手・野手カードの見出しとハイライト各行に `<img class="avatar ...">` を差し込み。
- `style.css`: `.avatar`（円形）/`.avatar-card`（44px）/`.avatar-mini`（22px）を追加。`.card-head` を `align-items: center` に、`.card-id` で写真＋名前をまとめた。ハイライト行を flex 化。
- ローカル確認: 表示対象の写真23枚すべて正常読み込み（フォールバック0）。
