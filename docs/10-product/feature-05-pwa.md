# ⑤ PWA化（スマホでアプリ風に使える）実装メモ

2026-06-04 実装。ロードマップ⑤の「かんたん版（PWA）」を実装した記録。
※ オフライン機能は不具合のため同日中に取り下げた（下記「オフライン機能を取り下げた経緯」参照）。

## 何をしたか（ひとことで）

今のWebサイトを「スマホのホーム画面に置けるアプリ風」にした。App Store 申請は不要・追加費用なし。
GitHub Pages 公開のまま動く。

## できるようになったこと（現状）

- スマホのブラウザで開いて「ホーム画面に追加」すると、**アイコンが置けてタップで全画面起動**。
- PC（Chrome/Edge）でも「インストール」ボタンからアプリのように起動できる。
- ※ オフライン表示・キャッシュによる高速化は**入れていない**（下記の経緯による）。常にネットから最新を取得する。

## 追加・変更したファイル

| ファイル | 役割 |
|---|---|
| `manifest.json`（新規） | アプリ名・アイコン・全画面起動などの設定 |
| `icon-192.png` / `icon-512.png`（新規） | ホーム画面アイコン（紺背景＋野球ボール） |
| `icon-maskable-512.png`（新規） | Android用（角を自動で丸く切り抜く形式） |
| `apple-touch-icon.png`（新規, 180px） | iPhone/iPad のホーム画面アイコン |
| `sw.js`（新規→中身を差し替え） | 当初オフライン用だったが、現在は「古い登録を解除しキャッシュを掃除するだけ」の後始末用 |
| `index.html`（変更） | manifest の読み込み・テーマ色・Apple向けタグを追加。SW登録は廃止し「解除＆掃除」に変更 |

## 設計のポイント

- **パスはすべて相対**（`./` 始まり）。GitHub Pages はURLに階層が付くため、先頭スラッシュ禁止の方針に合わせた。
- **キャッシュ対策は従来どおり `?v=日付`** に一本化（`index.html` で `app.js?v=` / `style.css?v=` を更新）。サービスワーカーによるキャッシュは使わない。

## オフライン機能を取り下げた経緯（重要な教訓）

- 当初、オフライン対応として `sw.js`（サービスワーカー）でファイルや成績データをキャッシュしていた。
- その結果、手元サーバー停止時や一時的な通信失敗時に、
  **名前の対応表(`config/name_map.json`)を読めず選手名が英語のまま出る／古いキャッシュを掴んで「空っぽ・最終更新—」になる** 不具合が発生した。
- この種の「古いものを掴んだまま」問題は非エンジニアのユーザーには原因が分かりにくく、手で消すのも難しい。
  もともと本プロジェクトは `?v=日付` でキャッシュ事故を警戒してきた経緯もあり、サービスワーカーは割に合わないと判断。
- **対処**：`sw.js` を「自分の登録を解除し、全キャッシュを削除して画面を読み直す」だけの後始末用スクリプトに置き換え、
  `index.html` 側でも既存の登録解除とキャッシュ削除を行うようにした。これで既存ユーザーのブラウザも次回読み込みで自動的にきれいになる。

## 動作確認（2026-06-04）

- ローカル（`python app.py`）で 14選手すべてが**日本語名**で表示され、コンソールエラー無しを確認。
- 読み込み後、**サービスワーカー登録 0 件・残キャッシュ 0 件**になることを確認（後始末が効いている）。
- `manifest.json` がブラウザに読み込まれ、アプリ風（全画面・ホーム画面追加）が機能することを確認。

## やらなかったこと（今後の選択肢）

- オフライン対応：今回は取り下げた。やるなら「ネットがある時は必ず最新を取る（ネット優先）」設計に限定すべき。
- プッシュ通知：PWAでも一部可能だが手間が増えるため見送り。通知は②メール通知の方が現実的。
- 本物のアプリ（App Store / Google Play）：個人利用には不要のため見送り。
