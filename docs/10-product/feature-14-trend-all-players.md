# 実装メモ：⑭ 推移ページを全選手に広げる

2026-06-05 実装。⑧で作った大谷専用の「成績の推移」ページを、全選手で見られるようにした。

## 1. 何をしたか（ひとことで）

- ⑧の大谷専用ページ（`ohtani.html` / `ohtani.js`）を**全選手共通の汎用ページ**に作り替えた。
- 一覧画面の**全選手のカード**に「📈 推移を見る」ボタンを出すようにした（⑧では大谷だけだった）。

## 2. 採用した方式と理由

- **その場取得（⑧の大谷方式を流用）**を採用。
  - 推移ページは開いた時にMLBへ「その選手の試合ごとログ」を問い合わせる（投手＋野手で2回）。
  - 「選手を1人選んで開く」形なので、一度に全選手を取りに行くわけではなく軽い。MLBにも優しい。
- ロードマップ当初は「毎朝ためる方式」を推していたが、それは“全選手を一度に取る”前提の心配だった。
  1人ずつ開く今回の作りなら、その場取得で十分。実装も軽く済んだ。

## 3. 変更したファイル

| ファイル | 変更内容 |
|---|---|
| `app.js` | `trendButton()` を全選手向けに変更。`ohtani.html?id=選手ID&tab=種別&name=日本語名` を付与。 |
| `ohtani.js` | 固定の大谷ID（`OHTANI_ID`）を廃止し、URLの `?id=` から選手IDを受け取る（無指定は大谷=660271）。日本語名は `?name=` から受け取り、無ければMLBから1回だけ補う。見出し・顔写真・タイトルを選手に合わせて差し替え。投手・野手タブをその選手の記録に応じて出し分け。 |
| `ohtani.html` | 見出し・顔写真・選手名にJSが差し替えるための目印（`id="page-title"` / `id="player-avatar"` / `id="intro-name"`）を追加。`ohtani.js?v=` のキャッシュ印を更新。 |
| `index.html` | `app.js?v=` のキャッシュ印を更新（2026-06-05）。 |

※ファイル名は `ohtani.html` のまま据え置き（既存リンク・ブックマークを壊さないため）。中身は全選手共通。

## 4. 仕様の要点

- **入り口**：各カードの「📈 推移を見る」ボタン → `ohtani.html?id=…&tab=…&name=…` を開く。
- **タブの出し分け**：
  - 投手専任 → 投手タブのみ（タブ바ー自体を隠す）。
  - 野手 → 野手タブのみ（同上）。
  - 二刀流（大谷）→ 投手・野手の両タブを表示。
  - どちらの記録も無い選手 → 「今シーズンの記録はまだありません。」と表示。
- **表示項目は⑧と共通**：
  - 野手：打率 AVG／本塁打 HR／打点 RBI／OPS／盗塁 SB
  - 投手：防御率 ERA／奪三振 K／投球回／WHIP
  - 各項目を上下2段（上＝シーズン累計の折れ線／下＝1試合ごとの棒）。下部に直近試合の表。
- **名前**：一覧データ（`data/latest.json`）が持つ日本語名を `?name=` で渡す。直接URLを叩かれて名前が無い時だけ、MLBの `/people/{id}` から英語名を1回補う。
