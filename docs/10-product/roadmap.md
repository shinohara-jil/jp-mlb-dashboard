# 改修計画（進化ロードマップ）

日本人メジャーリーガー成績ダッシュボードを、これからどう進化させていくかの計画メモ。
2026-06-02 時点の検討内容。難易度はあくまで目安（★が多いほど大変）。

## 前提（今の作り）

- GitHub Pages で公開している静的サイト。リポジトリはパブリック。
- データ取得はブラウザ側（app.js）が MLB 公式 API から直接行う。
- 毎朝 日本時間10時ごろ、GitHub Actions が `fetch_stats.py` を実行して `data/` をコミット。
- 画面は `data/latest.json` を読んで即表示。「🔄 更新」ボタンは直接 MLB からライブ取得。

## それぞれの詳細

### ① 画像も入れたい ★☆☆
- MLB の外部サービスが、選手の顔写真・チームのロゴを画像URLとして公開している。それを画面に貼るだけ。
- 新しい仕組みは不要。すぐ着手できるレベル。
- 「最新試合」カードに顔写真が並ぶと一気に華やかになる。最初の一歩におすすめ。

### ② メールで届くようにしたい ★★☆
- すでに「毎朝自動で成績を取りに行く」処理が動いているので、そこに「メールを送る」ステップを追加するだけ。
- 必要なもの：メール送信サービスの登録（無料枠あり）と、送り先のアドレス。
- 注意点：リポジトリは公開なので、メールのパスワード等の秘密情報は GitHub Secrets に隠して使う。

### ③ 更新タイミングを賢く ★★☆
- 簡単版：今は朝1回だけ → 1日に数回（朝・夕・夜など）動かす。設定の数字を変えるだけ。
- こだわり版：「その日の全試合が終わったら更新」。試合終了を判定する仕組みが要るので少し工夫が必要。
- まずは簡単版で「1日3回」くらいから始めるのが現実的。

### ④ 予想を入れたい ★★☆〜★★★
- 次の登板予定日・予告先発：MLB が公式に発表しているデータがあるので比較的かんたん（★★☆）。
- 次の試合の成績予想：本格的な「予測」。当たるかは別問題で、作り込むと難しい（★★★）。
  簡単にやるなら「シーズン平均からこのくらいでは？」という目安表示にとどめる。
- おすすめは「次の登板日・予告先発」から。

### ⑤ スマホアプリにしたい ★☆☆〜★★★
- かんたん版（PWA）：今のサイトを「スマホのホーム画面に置けるアプリ風」にする。申請不要・追加費用なし（★☆☆）。
- 本格版（App Store / Google Play）：開発者登録（Appleは年間約1.5万円）・審査・作り直しが必要で大変（★★★）。
- 個人で楽しむなら PWA で十分。まずこちらがおすすめ。

## おすすめの進め方（やりやすい順）

1. ① 画像（すぐ見栄えが良くなる・簡単）← **まずここから着手**
2. ⑤ PWA化（スマホで使いやすく・簡単）
3. ② メール通知 ＋ ③ 1日複数回更新（セットで相性が良い）
4. ④ 次の登板日表示（データがあるので現実的）
5. こだわり版（全試合終了判定／成績予想）は、上が落ち着いてから
