# 改修計画（進化ロードマップ）

日本人メジャーリーガー成績ダッシュボードを、これからどう進化させていくかの計画メモ。
2026-06-02 時点の検討内容。難易度はあくまで目安（★が多いほど大変）。

## 前提（今の作り）

- GitHub Pages で公開している静的サイト。リポジトリはパブリック。
- データ取得はブラウザ側（app.js）が MLB 公式 API から直接行う。
- 毎朝 日本時間10時ごろ、GitHub Actions が `fetch_stats.py` を実行して `data/` をコミット。
- 画面は `data/latest.json` を読んで即表示。「🔄 更新」ボタンは直接 MLB からライブ取得。

## それぞれの詳細

### ① 画像も入れたい ★☆☆ ✅ 実装済み・公開済み（2026-06-02）
- 選手の顔写真を、カード左（丸）とハイライト欄に表示。外部URL方式（追加ファイルなし）。
- 詳細・実装メモは [feature-01-images.md](feature-01-images.md) を参照。

### ② メールで届くようにしたい ★★☆
- すでに「毎朝自動で成績を取りに行く」処理が動いているので、そこに「メールを送る」ステップを追加するだけ。
- 必要なもの：メール送信サービスの登録（無料枠あり）と、送り先のアドレス。
- 注意点：リポジトリは公開なので、メールのパスワード等の秘密情報は GitHub Secrets に隠して使う。

### ③ 更新タイミングを賢く ★★☆
- 簡単版：今は朝1回だけ → 1日に数回（朝・夕・夜など）動かす。設定の数字を変えるだけ。
- こだわり版：「その日の全試合が終わったら更新」。試合終了を判定する仕組みが要るので少し工夫が必要。
- まずは簡単版で「1日3回」くらいから始めるのが現実的。

### ④ 予想を入れたい ★★☆〜★★★
- 次の登板予定日・予告先発：MLB が公式に発表しているデータがあるので比較的かんたん（★★☆）。
- 次の試合の成績予想：本格的な「予測」。当たるかは別問題で、作り込むと難しい（★★★）。
  簡単にやるなら「シーズン平均からこのくらいでは？」という目安表示にとどめる。
- おすすめは「次の登板日・予告先発」から。

### ⑤ スマホアプリにしたい ★☆☆〜★★★ ✅ PWA版 実装済み（2026-06-04）
- かんたん版（PWA）：今のサイトを「スマホのホーム画面に置けるアプリ風」にする。申請不要・追加費用なし（★☆☆）。**← 実装済み**
- 本格版（App Store / Google Play）：開発者登録（Appleは年間約1.5万円）・審査・作り直しが必要で大変（★★★）。**← 未実施（個人利用には不要のため見送り）**
- 実装内容：`manifest.json`（アプリ名・アイコン・全画面起動）／アイコン画像を追加。
- ⚠️ **オフライン機能（サービスワーカー）は取り下げた**：古いデータをブラウザが掴んだまま「空っぽ・英語名・更新日が—」になる不具合が出たため。`sw.js` は「古い登録を解除して掃除するだけ」の中身に置き換え済み。
  詳細・実装メモは [feature-05-pwa.md](feature-05-pwa.md) を参照。

### ⑥ リーグでソート（並べ替え）したい ★★☆
- MLB は「ア・リーグ（American League）」と「ナ・リーグ（National League）」の2リーグに分かれている。
  選手をリーグ別に並べ替え・絞り込みできるようにしたい。
- 必要なこと：各選手に「どちらのリーグか」の情報を持たせる。
  - 今の保存データにはチーム略称しか無いので、毎朝の取得（fetch_stats.py）で
    チーム→リーグの対応を引いて、各選手に `league`（AL／NL）を1項目足す軽い改修が必要。
  - 画面側（app.js）に「リーグで並べ替え／絞り込み」のボタンや見出しを追加する。
- 難易度は★★☆。データに1項目足す＋画面に切り替えUIを足す、という構成。
- 補足：表示方法は「投手・野手の中をさらにリーグ順に並べる」「リーグで絞り込むボタン」「リーグごとに見出しで区切る」などの選択肢がある（着手時に詰める）。

### ⑦ 試合中のデータに「試合中」タグを付けたい ★★☆
- 今は「最新試合」として直近1試合の成績を出しているが、その試合が**まだ進行中**の場合、
  表示している数字は「途中経過」になる。それを見分けられるよう「🟢 試合中」などの目印を付けたい。
- 必要なこと：その試合が「進行中か／終了済みか」を判定する。
  - MLB のデータには試合の状態（`Preview`＝開始前／`Live`＝進行中／`Final`＝終了）がある。
    今すでに使っている試合情報の取得（schedule）から、この状態も一緒に読めば判定できる。
  - 進行中なら、カードやハイライトに「試合中」タグを表示する（成績は途中経過の注記つき）。
- 難易度は★★☆。判定用の状態を1つ読み取り、画面にタグを足す構成。
- 補足：fetch_stats.py（毎朝の取得）とブラウザ側のライブ取得（🔄更新）の両方に同じ判定を入れると、
  どちらの経路でも「試合中」が正しく出る。着手時に詰める。

### ⑧ 大谷翔平だけ成績の推移を見たい（別ページ） ★★☆〜★★★
- 大谷翔平専用の別ページを作り、**今シーズンの成績の推移**を折れ線グラフなどで見られるようにする。
- **投手と野手でそれぞれ**表示（二刀流なので両方）。
  - 投手：防御率・奪三振・イニングなどの推移。
  - 野手：打率・本塁打・打点・OPS などの推移。
- データ：MLB API の試合ごとログ（gameLog）を日付順に並べれば推移が作れる。
  - 「今シーズンだけ」で一旦OK。過去シーズンとの比較は後回しでよい。
- 必要なこと：グラフ表示の仕組み（軽い描画ライブラリ or 自前の簡易グラフ）と、大谷専用ページの追加。
- 難易度は内容次第で★★☆（数字を並べる表＋簡単な棒グラフ）〜★★★（本格的な折れ線グラフ）。
- **2026-06-04：要件確定**（qaスキルでヒアリング済み）。詳細は [feature-08-ohtani-trend.md](feature-08-ohtani-trend.md)。
  - 投手・野手をタブ切替／今季のみ／各項目を上下2段（上＝累計の折れ線・下＝1試合ごとの棒）／大谷カードにボタン。

### ⑨ 直近のハイライト動画を再生したい ★★☆
- 今の「ハイライト欄」は成績の文字情報だけ。ここをクリックしたら、その試合の**ハイライト動画**を見られるようにしたい。
- データ：MLB には試合ごとの動画（ハイライト）を提供する仕組みがある（公式の動画コンテンツAPI）。
  - 試合IDから関連動画を引いて、再生リンクや埋め込みプレーヤーを表示する。
- 注意点：動画が必ずある試合・ない試合がある。無い場合の出し分けが要る。
- 難易度は★★☆。動画情報を1つ引いて、画面にリンク／プレーヤーを足す構成。

### ⑩ プレーオフに行けそうか分かるようにしたい（予想） ★★☆〜★★★
- 各選手の**所属チームがプレーオフに進出できそうか**を、予想として表示したい。
- かんたん版（★★☆）：チームの順位・勝率・地区順位を出すだけ（「今◯位」が分かる）。MLB API に順位表データあり。
- こだわり版（★★★）：「進出できる確率」の予想。残り試合数やマジックナンバー等を計算する必要があり作り込みが大変。
- おすすめは「順位・勝率の表示」から。確率予想は後回し。

### ⑪ 狙えそうな記録・タイトルを表示したい ★★★
- その選手が**狙えそうな記録やタイトル**（例：本塁打王に近い／200奪三振が見えてきた、など）があれば「あと少し」と教えたい。
- 必要なこと：
  - リーグ全体のランキングデータを取得し、上位に近い項目を見つける。
  - キリの良い数字（節目）まであと何本／何個か、を計算する。
- 「何をもって狙えるとするか」の線引き（ランキング上位◯位以内、節目まで残り◯など）を決める必要がある。
- 難易度は★★★。ランキング取得＋判定ルールの設計が要るため、着手時にしっかり詰める。

### ⑫ 怪我人の状態・復帰予定を表示したい ★★☆
- 今は「今シーズンの成績が無い選手は非表示」なので、**怪我で離脱中の選手が画面から消える**。
  そうではなく「怪我で離脱中」「復帰予定◯◯」と分かるように表示したい。
- データ：MLB には故障者リスト（IL＝Injured List）の情報や選手の状態（active/injured 等）がある。
  - 選手の状態を読み取り、怪我中なら「🤕 故障者リスト」タグ＋（あれば）復帰見込みを表示。
- 難易度は★★☆。状態を1つ読み取り、非表示にせずタグを付けて出す構成。
- 補足：復帰予定日は公式に必ず出るとは限らない。出ない場合は「離脱中」とだけ表示する。

### ⑬ 年俸を表示したい ★★☆〜★★★
- 各選手の**年俸（年間の契約金額）**をカードなどに表示したい。
- ⚠️ **ここが他と違う点**：年俸は**今使っているMLB公式API（無料）には含まれていない**。別の入手方法が必要。
  - 案A（かんたん・現実的／★★☆）：年俸をまとめた**自前の対応表**を1つ用意する（`config/salary.json` のようなファイルに「選手→年俸」を手入力）。
    選手は数十人なので現実的。ただし**毎年シーズンごとに手で更新**が要る。出典（Spotrac等の公開情報）を明記しておく。
  - 案B（自動・大変／★★★）：年俸を載せている外部サイト（Spotrac, Cot's Contracts 等）から自動取得する。
    ただし多くは公式API提供がなく、利用規約・取得の安定性・著作権の確認が必要で難易度が高い。
- おすすめは案A（手作りの対応表）。まず数字を出して、自動化は必要になってから検討。
- 表示方法：カードに「💰 年俸 ◯◯万ドル」のように添える（日本円換算を併記するかは着手時に相談）。

### ⑭ 推移ページを全選手に広げたい ★★☆ ✅ 実装済み（2026-06-05）
- ⑧で作った「成績の推移」ページを、大谷以外の選手でも見られるようにした。
- **採用した方式：その場取得（⑧の大谷方式を流用）**。
  - 「選手を1人選んで、その人のページを開く」形にしたので、問い合わせは開いた1人分の2回（投手＋野手）だけ。
    全選手を一度に取りに行くわけではないので軽く、MLBにも優しい。
  - ロードマップ当初は「毎朝ためる方式」を推していたが、それは“全選手を一度に”取る前提の心配だった。
    1人ずつ開く今回の作りなら、その場取得で十分（実装も軽く済んだ）。
- **入り口**：全選手のカードに「📈 推移を見る」ボタンを出す（⑧で大谷だけだったものを全員に解放）。
- **作り**：
  - ⑧の `ohtani.html` / `ohtani.js` を**全選手共通の汎用ページに作り替え**た（ファイル名は据え置き）。
    - 選手は URL の `?id=選手ID` で受け取り（無指定なら大谷）。日本語名は `?name=` で一覧から渡す。
    - 見出し・顔写真・ページタイトルを、選んだ選手に合わせて差し替え。
    - 投手・野手タブは**その選手に記録がある側だけ表示**（投手専任は投手だけ／野手は野手だけ／二刀流の大谷は両方）。
      片方しか無ければタブ自体を隠す。
  - `app.js` の「推移を見る」ボタンを全選手向けに変更（`?id=&tab=&name=` を付与）。
- **動作確認済み**：鈴木誠也（野手のみ）／今井達也（投手のみ）／大谷（両タブ）で表示・グラフ・タブ出し分けを確認。
- 表示項目は⑧と共通（野手＝打率・HR・打点・OPS・盗塁／投手＝防御率・奪三振・投球回・WHIP）。
  先発・中継ぎ共通でまず提供（セーブ等は必要になってから検討）。
- **2026-06-05：実装・確認完了**。
