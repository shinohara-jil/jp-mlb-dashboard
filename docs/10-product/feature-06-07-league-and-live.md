# 要件定義書：⑥リーグでソート（絞り込み）＋⑦試合中タグ

日本人メジャーリーガー成績ダッシュボードへの追加機能。2026-06-03 に要件化。
ベースの全体要件は [requirements.md](requirements.md)、進化計画は [roadmap.md](roadmap.md) を参照。

## 1. 概要

- **⑥リーグ絞り込み**：選手を「ア・リーグ（AL）／ナ・リーグ（NL）」で絞り込めるボタンを追加する。
- **⑦試合中タグ**：直近の試合がまだ進行中のとき、その選手のカードに「🟢 試合中」の目印を付ける。

## 2. 目的・背景

- ⑥：MLBは2リーグ制。「ア・リーグの選手だけ見たい」といった見方をしやすくしたい。
- ⑦：今は直近1試合の成績を出しているが、試合が進行中だと数字は「途中経過」。
  それを終了済みの確定成績と見分けられるようにし、誤解を防ぎたい。

## 3. 対象ユーザー

- 作成者本人。公開範囲は GitHub Pages（パブリック）だが、利用者は本人想定。

## 4. 機能要件

### ⑥-1. リーグ情報の付与

- 各選手に「所属リーグ（AL／NL）」の情報を持たせる。
- 毎朝の自動取得（`fetch_stats.py`）で、チーム → リーグの対応を引いて
  各選手に `league`（"AL" / "NL"）を1項目追加して保存する。
- チーム→リーグの対応は **固定の対応表**（30球団分）を設定として持つ（`config/` に置く想定）。
- リーグが判定できない場合は空（絞り込み対象外）として扱い、エラーにしない。

### ⑥-2. リーグ絞り込みボタン

- 画面に「全部／ア・リーグ／ナ・リーグ」の**切り替えボタン**を置く。
- 選んだリーグに所属する選手だけを表示する（投手・野手の区分はそのまま維持）。
- 初期表示は「全部」。
- 大谷選手（二刀流）は所属リーグ（NL／Dodgers）の絞り込みに従い、投手・野手の両方に出る。

### ⑦-1. 試合状態の判定

- 直近試合が「進行中（Live）か／終了済み（Final）か／開始前（Preview）か」を判定する。
- MLBデータの試合状態（Preview＝開始前／Live＝進行中／Final＝終了）を読み取って判定する。
- 判定は **毎朝の自動取得（`fetch_stats.py`）と、画面の「🔄 更新」ボタンによるライブ取得の両方**に入れる。
  どちらの経路でも「試合中」が正しく出るようにする。

### ⑦-2. 試合中タグの表示

- 直近試合が進行中のとき、その選手の**最新試合カード**に「🟢 試合中」タグを表示する。
- **ハイライト欄の各行（選手名の横）にも同じタグを表示する**（2026-06-04 仕様変更で追加）。
- 成績の数字そのものへの注記（「（途中経過）」等）は付けない。タグのみで表現する。

## 5. 画面構成

- 既存の1画面ダッシュボードを踏襲。追加は以下のみ。
  - 上部（投手・野手セクションの近く）に「全部／ア・リーグ／ナ・リーグ」の絞り込みボタンを追加。
  - 進行中の試合をしている選手の最新試合カードに「🟢 試合中」タグを表示。
- スマホ・PCのどちらでも崩れないようにする。

## 6. 非機能要件

- 動作環境：Webブラウザ（GitHub Pages 公開版／ローカル確認版の両方）。
- パスは相対パスを維持（GitHub Pages の階層対応）。
- リーグ判定・試合状態判定が取れない場合もエラーにせず、従来どおり表示する（タグ無し・絞り込み対象外）。

## 7. 制約・注意事項

- リーグの対応表（チーム→AL/NL）は固定の対応表（30球団分）を `config/` に持つ。
  将来エクスパンション等で球団が増えた場合のみ手直しが必要（当面は変わらない）。
- 試合状態は時間によって変わるため、「🔄 更新」を押した瞬間の状態を表示する。
  毎朝の自動取得時点（日本時間10時ごろ）は多くの試合が終了済みのため、タグが付くのは主にライブ更新時。
- データ項目を増やす（`league`・試合状態）ため、`data/latest.json` の形が少し変わる。

## 実装メモ（2026-06-03 実装・確認済み）

### ⑥ リーグ絞り込み
- `config/team_league.json` … 30球団の略称→AL/NL 固定対応表（略称はMLB公式 `abbreviation` に一致）。先頭 `_comment` キーは説明用で処理時に除外。
- `fetch_stats.py` … `load_team_league()` を追加。`build_player_data` で各選手に `league`（"AL"/"NL"、不明なら空）を付与。
- `app.js` … `loadTeamLeague()` を追加。`buildPlayer` で `league` を付与。画面に絞り込み状態（`currentLeague`）を持ち、ボタン押下で `render` を再実行して絞り込む。絞り込みはハイライト・投手・野手すべてに適用。
- `index.html` … 「全部／ア・リーグ／ナ・リーグ」ボタン（`#league-filter`）を追加。
- `style.css` … 丸いピル型ボタン（`.filter-btn` / `.is-active`）。

### ⑦ 試合中タグ
- 試合状態は MLB schedule の `status.abstractGameState`（`Preview`/`Live`/`Final`）で判定。`Live`＝試合中。
- `fetch_stats.py` … 既存の試合日取得を `get_game_info()` に拡張し、日付＋`live`（試合中か）を返す。`fetch_latest_game` の戻り値に `live` を追加。
- `app.js` … 同様に `getGameInfo()` へ拡張、`fetchLatestGame` に `live` を追加。`liveTag()` で「🟢 試合中」をカードの「最新登板／最新試合」見出し横に表示。**ハイライト欄の各行（選手名の横）にも `liveTag()` を表示**（2026-06-04 追加）。数字への注記は無し。
- `style.css` … 緑のピル型タグ（`.live-tag`）。
- 毎朝の自動更新（fetch_stats.py）と🔄ボタン（app.js）の両経路に同じ判定を実装済み。朝の更新時はほぼ試合終了のためタグは付かず、主にライブ更新時に表示される。

### さわったファイル
- 新規: `config/team_league.json`
- 変更: `fetch_stats.py` / `app.js` / `index.html` / `style.css`
- データ: `data/latest.json` に各選手 `league` と最新試合 `live` が追加された

### 確認結果
- ローカルサーバー（`python app.py`）＋ブラウザで動作確認済み。
  - 絞り込み: 全部=投手9/野手6、AL=投手2/野手4、NL=投手7/野手2（合計が一致）。
  - 試合中タグ: 進行中試合が無い状態ではタグ0。擬似的に `live=true` にすると「🟢 試合中」がカードに表示されることを確認。
