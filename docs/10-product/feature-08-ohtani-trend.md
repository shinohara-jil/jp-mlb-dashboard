# 要件定義書：⑧ 大谷翔平 成績推移ページ

日本人メジャーリーガー成績ダッシュボードへの機能追加。
2026-06-04 にヒアリング（qaスキル）で確定した内容。

## 1. 概要

- 大谷翔平**専用**の「成績の推移」を見る**別ページ**を新設する。
- 今シーズンの成績が、試合を重ねるごとにどう動いてきたかをグラフで振り返れるようにする。
- 大谷は二刀流なので、**投手・野手の両方**の推移を見られるようにする。

## 2. 目的・背景

- 今の一覧画面は「最新試合」と「シーズン通算」の“今の数字”しか分からない。
- 「ここ最近調子が上がってきた／本塁打が積み上がってきた」といった**流れ（推移）**が見たい。
- まずは注目度の高い大谷翔平だけを対象に、推移ページの形を作る。

## 3. 対象ユーザー

- このダッシュボードを見る人（非エンジニアの個人利用が中心）。

## 4. 機能要件

### 4-1. 対象・期間
- 対象選手：**大谷翔平のみ**。
- 期間：**今シーズンのみ**（過去シーズンとの比較は対象外）。
- データ取得元：既存と同じ **MLB公式 StatsAPI**。試合ごとログ（gameLog）を日付順に並べて推移を作る。

### 4-2. 見せる項目
- **投手**：防御率(ERA)／奪三振(K)／イニング・勝敗／WHIP
- **野手**：打率(AVG)／本塁打(HR)／打点(RBI)／OPS／盗塁(SB)

### 4-3. 推移の見せ方（重要・画面イメージ確定）
- 各項目について、**上下2段**で表示する（レイアウトB）。
  - **上段＝「移り変わり（シーズン累計）」の折れ線グラフ**
    - 試合を重ねるごとの“その時点でのシーズン数字”を線でつなぐ。
    - 例（HR）：累計本数 1→2→4→6→8 と積み上がる線。
    - 例（AVG）：その時点での通算打率 .280→.295→.290… と動く線。
  - **下段＝「1試合ごと」の棒グラフ**
    - その試合の数字を1本ずつ棒で出す。
    - **数えるもの**（HR・RBI・SB・K・イニング）：その試合の本数／個数をそのまま棒にする。
    - **割合のもの**（AVG・ERA・OPS・WHIP）：割合そのものは1試合だと極端に振れるため、
      **“元になる数”を棒で出す**。
      - 例：打率 → その試合の**安打数**を棒で。
      - 例：防御率 → その試合の**自責点**を棒で。
      - 例：OPS → その試合の**出塁＋長打に関わる数**（着手時に具体項目を確定）。
      - 例：WHIP → その試合の**与四球＋被安打**（着手時に具体項目を確定）。
- グラフの下に、**試合ごとの数字の表を少しだけ**添える（日付・対戦相手・その試合の成績・累計）。

### 4-4. 入り口（ページへの行き方）
- 一覧画面の**大谷のカードに「推移を見る」ボタン**を置き、押すと推移ページへ移動する。

### 4-5. 投手・野手の切り替え
- 推移ページ内で「**投手**」「**野手**」を**タブで切り替え**て表示する。

## 5. 画面構成

### 一覧画面（既存）
- 大谷のカード（投手側・野手側どちらでも可。着手時に置き場所を確定）に
  「📈 推移を見る」ボタンを追加。

### 推移ページ（新規）
- 上部：ページ見出し（例「大谷翔平 シーズン推移」）＋ 一覧に戻るリンク。
- タブ：［ 投手 ］［ 野手 ］
- 各項目ごとに2段グラフ：
  - 上段：折れ線（移り変わり＝シーズン累計）
  - 下段：棒（1試合ごと）
- 下部：試合ごとの数字の小さな表。

```
┌──────────────────────────────────┐
│ 大谷翔平 シーズン推移      ← 一覧に戻る │
│  [ 投手 ] ( 野手 )                  │
├──────────────────────────────────┤
│ 【本塁打 HR】                       │
│ ▲ 移り変わり（累計）   折れ線        │
│ ▼ 1試合ごと（その日の本数） 棒       │
│ 【打率 AVG】                        │
│ ▲ 移り変わり（通算打率） 折れ線      │
│ ▼ 1試合ごと（その日の安打数） 棒     │
│  …（他項目も同じ形で続く）           │
│ ── 試合ごとの表（日付/相手/成績/累計）│
└──────────────────────────────────┘
```

## 6. 非機能要件

- 動作環境：PCブラウザ中心（既存サイトと同じ。スマホでも崩れない配慮はする）。
- 公開範囲：既存と同じく **GitHub Pages で一般公開**（リポジトリはパブリック）。
- グラフ表示には軽量な無料のグラフ表示部品を1つ使う想定。
  **オンライン前提**（既存サイトもオンライン前提のため問題なし）。
- パスは**相対パス**で参照（先頭スラッシュ禁止）。サイトファイルはリポジトリ直下に置く。
- キャッシュ対策：新規ファイル（追加するJS/CSS）には既存ルールに従い `?v=日付` を付ける。

## 7. 制約・注意事項

- 今シーズンの試合数がまだ少ない時期は、グラフの点が少ない（仕様上問題なし）。
- 大谷が故障等で今シーズンの該当成績が無い場合の表示（投手だけ／野手だけになる等）は着手時に確認。
- 割合系（AVG/ERA/OPS/WHIP）の下段「元になる数」の具体項目は、着手時（設計段階）に最終確定する。
- まずは大谷専用。将来、他選手にも横展開できる作りにできると望ましい（必須ではない）。
